### 더 좋은 코드를 만들려면 어떻게 해야 할까요?
* 우선 좋은 코드는 처음 본 사람도 내용을 어렵지 않게 이해할 수 있는 잘 정리 되어있는 코드라고 생각합니다.
* 우선 코드에서 특정 영역마다 주석을 넣어서 그것이 어떤 변수인지 혹은 어떤 방식으로 넣는 것인지는 필수적으로 설명해야 한다고 생각합니다.
* 변수나 함수는 역할과 관련된 이름으로  `Lower camel case` 방식으로 명명을 합니다.
* 그리고 공백을 적절히 사용하여 코드의 가독성을 높입니다.


### 더 좋은 코드는 끊임없이 바뀌는 요구사항들에 겪는 고통을 최소한으로 줄일 수 있는 코드입니다.

#### 만약 완성된 코드에서 두 개의 데스크탑을 따로 두어 실행되도록 하려면 어떻게 해야 할까요?
* 완성된 코드에서 Desktop 객체는 선택자를 사용해서 이용하도록 만들었습니다. 그러므로 서로 데스크톱으로 이용할 HTML 태그를 두 개 만들어서 크기를 조정하고 서로 다른 class 혹은 id를 놓고 실행시키면 될 것 입니다.

#### 만약 시작할 때 주어지는 세팅(한 개의 일반 아이콘, 두 개의 폴더 아이콘)을 자유롭게 바꾸고 싶다면 어떻게 해야 할까요?
* Desktop 객체의 생성자를 넣을 때 아이콘 css 형식의 선택자 외의 2개의 매개변수를 넣도록 작성했습니다. 첫 번째는 일반 아이콘, 두 번째는 폴더 아이콘의 갯수를 설정하는 변수입니다.
* 세팅을 자유롭게 바꾸고 싶을 때, 그 뒤에 일반 아이콘과 폴더 아이콘의 갯수를 넣으면 될 것 입니다.

#### 아이콘의 모양과 크기가 언제든지 바뀔 수 있다면 어떻게 해야 할까요?
* Desktop 객체의 함수 중에 `setIconSize`, `setIconImage`, `setFolderImage` 메서드로 아이콘의 모양과 크기를 변경할 수 있게 만들었습니다.
* `setIconSize`는 width, height를 조정하는 함수로 매개변수를 하나만 넣으면 width, heigth 둘다 같은 값이 적용됩니다.
* `setIconImage`, `setFolderImage` 메서드는 각각 일반 아이콘, 폴더 아이콘의 이미지를 변경하는 함수입니다.
